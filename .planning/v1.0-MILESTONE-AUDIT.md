## Critical Gaps

### 1. Illinois packet is not publicly reachable through the site shell

This is the milestone blocker.

The site now describes itself as a `model / argument / proof` product:
- model = Illinois packet
- argument = financing explainer
- proof = registry

But the built shell only exposes:
- thesis
- methodology
- registry catalog
- registry detail pages
- explainer

The model layer is described but not actually reachable.

### Why this matters

- It breaks the most important public reading path.
- It collapses the distinction between the normative packet and descriptive Illinois registry record.
- It makes the homepage and explainer CTAs materially misleading.

## Tech Debt

### Phase 5 tooling debt

- `bun run format` fails because Biome is not configured to parse Tailwind v4 CSS directives in [src/styles/app.css](/Users/peterryszkiewicz/.codex/worktrees/8fd7/bitcoin-bond-proposal/src/styles/app.css).
- `bun run lint` checks generated `.output/public` artifacts and emits thousands of non-source diagnostics, which reduces the usefulness of the lint gate.

These do not block milestone completion by themselves, but they should be cleaned up before the next milestone expands the public shell.

## Recommendation

Do **not** archive the milestone yet.

The next work should be a narrow gap-closure pass that does one of the following cleanly:
- add public packet routes for the Illinois one-pager and draft bill, or
- expose the packet PDFs directly from the public shell with honest labels, or
- relabel the existing CTAs so they stop claiming the Illinois registry page is the Illinois model packet

The strongest fix is to add a real public packet layer and keep the registry entry clearly separate.
