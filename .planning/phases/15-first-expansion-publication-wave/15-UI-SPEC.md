# Phase 15 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Tailwind CSS v4 with repo-local `@theme` tokens in `src/styles/app.css`; no shadcn |
| Preset | not applicable |
| Component library | Repo-local Solid editorial components only: `PageSection`, `StateCard`, `Badge`, `ActionLink`, `MarkdownContent`, `RegistryFreshnessPanel` |
| Icon library | none; do not add an icon package for Phase 15 |
| Font | Body: `"Avenir Next", "Segoe UI", "Helvetica Neue", sans-serif`; headings: `"Iowan Old Style", "Palatino Linotype", "Book Antiqua", Georgia, serif` |

Design source: existing SolidStart/Tailwind editorial system detected from `package.json`, `src/styles/app.css`, and `src/components/editorial/*`. `components.json` is absent, and the app already has a working Solid editorial component system, so the shadcn initialization gate is not applicable for this phase.

Phase 15 is a content-publication phase. Do not introduce route-specific UI components, new page templates, new component libraries, new icon libraries, or visual redesign work. Newly published records must render through the existing `/states/[slug]` route and existing state-card/catalog components.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline text/code padding and tight badge internals |
| sm | 8px | Badge padding, compact nav gaps, small vertical rhythm |
| md | 16px | Default panel/card padding, grid gaps, form-control spacing |
| lg | 24px | Detail aside padding, section dividers, stacked support panels |
| xl | 32px | Reading-surface padding on larger screens and larger content gaps |
| 2xl | 48px | Major route section breaks when an existing layout already uses them |
| 3xl | 64px | Page-level spacing only; do not add new Phase 15 page sections |

Exceptions: Preserve existing 20px `p-5`, `gap-5`, and `mt-5` spacing in established card/panel components such as `StateCard`; do not introduce new Phase 15-specific spacing exceptions. No icon-only touch-target exception is needed because Phase 15 does not add icon controls.

## Typography

Use exactly these four type sizes for any Phase 15 UI touch. Keep existing component typography unless a rendering defect requires a minimal fix.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / metadata | 12px | semibold 600 | 1.4 |
| Supporting UI text | 14px | regular 400 | 1.5 |
| Body / reading text | 16px | regular 400 | 1.5 |
| Section heading | 30px | semibold 600 | 1.2 |

Weights are limited to regular 400 and semibold 600. Existing serif headings, uppercase metadata labels, and state record copy should wrap naturally; do not add viewport-scaled font sizes or route-specific title treatments. Long bill titles, status strings, source names, and policy-effect summaries must remain readable without horizontal overflow at 375px and 1280px viewports.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0b0908` | Page canvas and dark editorial background |
| Secondary (30%) | `#1a1411` | Cards, reading surfaces, support panels, chips, nav surfaces |
| Accent (10%) | `#d98b3f` | Links, focus rings, active nav state, badge/chip emphasis, source-trail links, existing primary `ActionLink` background |
| Destructive | not used | Phase 15 has no destructive public UI action; do not introduce destructive controls |

Accent reserved for: Markdown links, source-trail links, active navigation pills, badge/chip emphasis text, focus-visible rings, and existing `ActionLink` primary/hover states. Do not use accent for broad backgrounds, new decorative gradients, state-specific color coding, or comparison/catalog polish deferred to Phase 17.

Semantic support colors already present in `src/styles/app.css` may remain: `#7ec49b` for existing reserve/success badge tone and `#d18a58` for existing warning-style cues. Do not add new semantic colors in Phase 15.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open state record |
| Empty state heading | No matching state records |
| Empty state body | Adjust the type filter or return to all published states. |
| Error state | This state record is not published. Try the published registry batch from the states index instead. |
| Destructive confirmation | None: Phase 15 does not add destructive public UI actions. |

New state copy must preserve the existing descriptive, source-first editorial voice. Status copy must be conservative and dated, using official-source posture rather than advocacy framing. Policy-effect summaries should explain what the proposal would do or did, not sell momentum. For Utah, copy must avoid implying a narrow Bitcoin reserve enactment when the official enrolled text is broader digital-asset/blockchain legislation. For failed records, copy must say failed, died, deferred, or inactive as supported by the official source.

## Phase Surface Contract

| Surface | Contract |
|---------|----------|
| `/states/[slug]` | Required render path for every newly published Phase 15 state. No slug-specific route branches, hard-coded imports, or alternate templates. |
| `/states` | May consume the expanded compiled graph through existing catalog/filter/sort behavior. Fix only data-contract breakage or text overflow caused by new records. |
| `/states/clusters` | No Phase 15 visual or copy polish unless expanded data breaks navigation or rendering. Broader cluster polish is Phase 17. |
| `/states/compare` | No Phase 15 visual or copy polish unless expanded data breaks navigation or rendering. Broader comparison polish is Phase 17. |
| Header/footer/nav | Preserve existing sticky dark editorial chrome. New state publication must not add nav items or alter global chrome. |

Required interaction checks after implementation:

- Each new state card/link opens the canonical `/states/[slug]` detail page.
- Detail pages preserve the existing desktop two-column layout and mobile stacked layout.
- Long status strings, bill IDs, source URLs, source names, and policy-effect summaries wrap inside their containers.
- Focus-visible rings remain visible on state cards, nav links, and source/action links.
- Not-found behavior remains unchanged for unpublished slugs.
