# Phase 8: Dark Theme Foundation - Context

## Decisions Captured

### Dark palette direction
- The site should be clearly dark by default, but not black-on-black.
- The foundation should use near-black editorial tones with two to three dark neutral layers so shell, cards, and rails separate cleanly.
- The overall palette should lean slightly warm rather than cold:
  - coal
  - espresso
  - graphite
- Reading surfaces should carry medium-high contrast:
  - strong readability for body text
  - softened enough panels to keep the site feeling authored rather than stark
- Backgrounds should use subtle depth:
  - restrained gradients
  - low-noise tonal shifts
  - calmer reading surfaces

### Orange accent intensity
- Bitcoin-orange should be restrained and strategic, not loud.
- It should show up most in:
  - primary CTAs
  - active nav states
  - eyebrows and key labels
  - selected proof and diagram highlights
  - focus and emphasis moments
- It should stay limited in:
  - large page backgrounds
  - large card fills
  - long text blocks
  - neutral status language
- The accent should be slightly muted rather than bright exchange-orange.
- The system should use multiple orange tones:
  - one stronger accent
  - one softer accent
  - one low-contrast wash

### Shared surface treatment
- Phase 8 should cover the shared system only:
  - tokens and variables
  - page background system
  - header and footer
  - buttons and links
  - cards and panels
  - badges and chips
  - dividers and borders
- Typography should shift lightly with the dark system through spacing, color, and hierarchy treatment.
- Shared diagrams and reusable editorial components should adopt the new token system in Phase 8.
- Methodology and registry fact surfaces should share the same dark system, but remain calmer than overtly persuasive editorial surfaces.

### Atmosphere versus seriousness
- The foundation should feel noticeably more atmospheric than v1.0, but still restrained.
- Acceptable atmosphere:
  - soft gradient depth
  - subtle metallic warmth
  - low-noise texture
  - quiet highlight edges
  - carefully placed orange emphasis
- Avoid:
  - neon orange glow
  - luminous outlines everywhere
  - full black/orange saturation as the whole theme
  - generic cyber styling
  - overly animated gradients
- Trust-first surfaces must remain visually disciplined:
  - methodology
  - status and freshness blocks
  - source trails
  - fact tables

## Deferred Boundary
- Route-by-route polish, readability QA, and mobile regression cleanup belong to Phase 9.
- Theme toggle work remains out of scope for this milestone.
