# Phase 999.1 Closeout Evidence

## Compatibility Gate

Phase 999.1 treated Mystic UI adoption as evidence-gated, not automatic. The current public app is SolidStart with Tailwind CSS v4 via `@tailwindcss/vite` and CSS-first repo tokens in `src/styles/app.css`, while the observed Mystic UI source package path remains documented around a Vite + SolidJS + Tailwind CSS 3.x setup and a broader theme/dependency surface.

Mystic UI was locally adapted; the direct package was rejected for now because it did not clear the repo's Tailwind CSS v4, SolidStart, token, and trust-surface gates.

### Direct Package Rejection

Direct package adoption is rejected for this phase because it would add package and lockfile churn for a narrow comparison/proof treatment, while the existing route can deliver the needed reader value through local Solid components, `panel-wash`, `data-chip`, and the current editorial token system.

This applies these phase decisions:

- D-01: adoption is evidence-gated, not automatic.
- D-02: only an exact current GitHub SHA would be eligible for a future direct adoption attempt.
- D-03: Tailwind CSS v4, token, dark-mode, or broad setup mismatch closes the direct package path for now.
- D-07: local adaptation is preferred when it delivers the proof/comparison value with less dependency risk.
- D-09: shared chrome, footer, metadata, README, profile, about, settings, and identity surfaces remain untouched.
- D-12: this evidence records that Mystic UI was locally adapted and the direct package was rejected for now.
