# Archived Requirements: v1.0

**Defined:** 2026-03-31  
**Archived:** 2026-04-03  
**Milestone:** v1.0 Illinois-first MVP

## v1 Requirements (Final Outcome)

### Illinois Package

- [x] **DOCS-01**: Reader can access a one-page Illinois summary that explains the reserve-and-bond proposal, rationale, and key mechanics in legislator-friendly language.
- [x] **DOCS-02**: Reader can access a longer-form Illinois draft bill structured as a serious first-pass legislative document.
- [x] **DOCS-03**: Reader can access a methodology memo that explains source hierarchy, freshness rules, interpretive stance, and publication criteria.
- [x] **DOCS-04**: Reader can see explicit caution language clarifying that the project is policy analysis, not legal or investment advice.

### Research Registry

- [x] **CATA-01**: Maintainer can maintain a 50-state skeleton catalog for reserve and bond legislation.
- [x] **CATA-02**: Reader can access 3-5 fully researched state entries in v1.
- [x] **CATA-03**: Reader can see whether each published entry concerns reserve policy, bond policy, or both.
- [x] **CATA-04**: Reader can see bill or law identifier, chamber, sponsors, status, and last action date for each published entry.
- [x] **CATA-05**: Reader can see a plain-English summary and a “what this would actually do” explanation for each published entry.
- [x] **CATA-06**: Reader can inspect primary source links plus visible `status as of` and `last reviewed` dates for each published entry.

### Publishing Pipeline

- [x] **PIPE-01**: Maintainer can build PDFs from canonical Markdown source documents.
- [x] **PIPE-02**: Pre-commit validation stops a commit if required PDF artifacts or related checks fail.
- [x] **PIPE-03**: PDF outputs remain derived from canonical Markdown so packet content and website content do not drift.

### Public Site

- [x] **SITE-01**: Reader can view a thesis-led homepage explaining why state Bitcoin reserves and Bitcoin-backed bonds matter.
- [x] **SITE-02**: Reader can view a methodology page in the public site.
- [x] **SITE-03**: Reader can browse a catalog page listing tracked state proposals.
- [x] **SITE-04**: Reader can open a state detail page template backed by canonical content.
- [x] **SITE-05**: Reader can read at least one deeper explainer article.
- [x] **SITE-06**: Reader can view a minimal but coherent static site shell that reflects the `free-the-world` research-registry influence without copying it rigidly.
