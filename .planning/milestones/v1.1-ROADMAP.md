# Milestone v1.1: Dark Mode Editorial Refactor

**Status:** ✅ SHIPPED 2026-04-04  
**Phases:** 8-9  
**Total Plans:** 4

## Overview

This milestone turned the public site into a dark-by-default editorial product without sacrificing the trust cues established in v1.0. It established a warm near-black theme foundation, introduced restrained Bitcoin-orange emphasis, then polished every shipped route for readability, interaction clarity, and mobile stability.

## Phases

### Phase 8: Dark Theme Foundation

**Goal**: Establish the dark-first token layer, shared shell styling, and reusable accent system for the webapp.  
**Depends on**: Phase 7  
**Plans**: 2 plans

Plans:
- [x] 08-01: Establish the dark-first token layer and root shell foundation.
- [x] 08-02: Adopt the dark system across shared editorial components.

**Details:**
This phase established the shared theme system and dark shell surfaces. It deliberately stopped short of route-by-route polish except where a hardcoded light blocker would have made the dark system look broken.

### Phase 9: Route Polish and Readability QA

**Goal**: Apply and polish the dark editorial system across all shipped routes, with explicit readability and interaction QA.  
**Depends on**: Phase 8  
**Plans**: 2 plans

Plans:
- [x] 09-01: Tune homepage readability and proof/catalog scanability.
- [x] 09-02: Polish long-form reading surfaces and complete route QA.

**Details:**
This phase preserved the existing route structure while sharpening hierarchy, improving long-form comfort, and fixing the remaining mobile and focus-state regressions discovered during runtime QA.

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Keep the dark system warm and editorial rather than cold, blue-black, or cyber-styled.
- Use restrained Bitcoin-orange emphasis on actions, nav states, labels, and selective proof surfaces rather than large saturated fills.
- Preserve the existing public reading structure instead of redesigning routes during the dark refactor.
- Treat mobile and focus-state fixes as part of the milestone definition of done for a visual refactor.

**Issues Resolved:**
- Replaced the prior light-first shell with a structurally dark public site.
- Eliminated route-level half-light artifacts across homepage, methodology, packet, explainer, catalog, and state-detail pages.
- Closed the mobile header overflow discovered during runtime QA.
- Restored visible focus treatment across navigation, actions, proof cards, and catalog cards in the dark shell.

**Issues Deferred:** None at milestone level.

**Technical Debt Incurred:** None at milestone level.
