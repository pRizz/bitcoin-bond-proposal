# Milestone v1.2: National Coverage, Comparison, and Refresh

**Status:** ✅ SHIPPED 2026-04-12  
**Phases:** 10-13  
**Total Plans:** 8

## Overview

This milestone turned the Illinois-first product into a broader national research surface.
It strengthened the canonical registry contract for grouping and refresh work, expanded
the published registry from five to ten state records, added dedicated cluster and
comparison reading paths, and finished with a repo-owned refresh queue plus shared
freshness cues across the public registry surfaces.

## Phases

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Keep refresh-only workflow artifacts outside canonical state content under `generated/refresh/`.
- Reuse one shared site-content layer for grouped views, comparison sections, and later
  refresh summaries instead of rebuilding logic in route code.
- Keep cluster and comparison routes editorial and source-linked rather than drifting
  into dashboard or matrix UI.
- Use one shared freshness threshold layer for both maintainer refresh workflow and
  public-facing trust cues.

**Issues Resolved:**
- Closed the structural gap between published content and future refresh artifacts.
- Expanded the registry beyond the first five states without weakening the source and
  freshness contract.
- Added grouped and comparison reading paths while preserving canonical detail-page
  drill-down.
- Added a repo-owned refresh queue and cross-surface freshness cues without introducing
  pseudo-live claims.

**Issues Deferred:**
- Browser-level regression automation for `/states` filters and grouped-route badge
  rendering remains a future hardening task.
- One resilience path in the shared content selector layer could still use a more
  focused unit test.

**Technical Debt Incurred:**
- Manual browser QA still carries part of the grouped-route regression burden.
